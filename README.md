# 考勤统计汇总系统

读取考勤机导出的 Excel，自动算工时、算午休扣除、处理跨天打卡，生成汇总报表和 Excel 导出。

后端 Spring Boot + 前端 React，数据存在 PostgreSQL 里。

---

## 页面功能

打开首页后顶部是统计卡片，下面分 7 个 Tab：

- **人员汇总**：按总工时从高到低排，一眼看出谁干了多少。点姓名直接跳到他本人的明细页。
- **个人明细**：左侧选人，右侧看这个人每天的打卡、工时、异常标记。
- **考勤汇总**：横向日历矩阵，每人一行、每天一格，当天工时是几就填几。没打卡的日子显示 `-`，工时特别高（≥10）的格子会泛绿，特别低（<2）的泛灰。点姓名也能跳明细。
- **全部明细**：所有人所有天的明细放一张大表，可以全局搜。
- **原表数据**：你上传的 Excel 原始长什么样，这里原样展示。
- **人员管理**：给每个人单独开三个开关——厂区住宿、弹性午休、晚餐扣除。开哪个勾哪个，报表自动重算。任杰、王清玲、梁宗茂三个人是系统默认标好的，页面里会蓝色高亮显示。
- **计算说明**：系统规则的备忘文档。

## 考勤是怎么算的

### 1. 从 Excel 里读什么

考勤机导出的文件通常有个 `考勤记录` sheet，第 5 行开始是人员信息（工号、姓名、部门），再往下隔一行是该人每天的原始打卡文本。系统会把这些原始文本里的 `HH:mm` 时间全部抽出来，连续重复的时间只留一个。

比如原文本是 `07:5107:51`，系统只认一次 `07:51`。

### 2. 跨天合并

有些同事加班到很晚，甚至次日凌晨才打下班卡。如果前一天有其他白天打卡（≥08:00），且次日凌晨的卡在 06:00 之前，系统会把凌晨这张卡**并回前一天**，按跨天加班处理。合并后前一天会标上「跨天并单」的异常标记。

**例子**：
- 4 月 1 日：`07:30` 上班，`12:00` 午休出门，`13:00` 午休回来，`22:00` 下班，然后 `02:30`（4 月 2 日凌晨）又打了一次
- 系统会把 `02:30` 这张卡并回 4 月 1 日，4 月 1 日实际打卡视为：`07:30`、`12:00`、`13:00`、`22:00`、`02:30`
- 4 月 2 日本身不再显示 `02:30`

### 3. 午休怎么扣

系统里每个人可以独立配置两个开关：「厂区住宿」和「弹性午休」，组合起来有四种情况：

**都不勾（普通）**
- 固定扣 `12:00-13:00`，扣 60 分钟
- 不管你中午有没有打卡，都扣 60 分钟

**只勾「厂区住宿」**
- 固定扣 `12:00-14:00`，扣 120 分钟

**只勾「弹性午休」**
- 默认也扣 `12:00-14:00`，扣 120 分钟
- 但如果 `12:00-14:00` 之间有打卡（比如 12:30 回来），按最早的午间打卡时间恢复计时，扣的时间会少于 120 分钟

**两个都勾**
- 同「只勾弹性午休」的逻辑，有午间打卡就按最早恢复

**晚餐扣除（单独开关）**
- 当天打卡次数 ≥4 时，额外再扣 60 分钟晚餐时间
- 这个开关默认只给任杰开了

### 4. 工时怎么算

**第一步：算原始时长**

最早打卡到最晚打卡之间的分钟数。

**第二步：扣午休和晚餐**

实际工时 = 原始时长 − 午休扣除 − 晚餐扣除

**第三步：换算成「工时」**

系统里的「工时」不是简单的小时数，而是按 20 分钟为单位取的：

```
工时 = floor(实际分钟 / 20) × 0.5
```

也就是说每满 20 分钟算 0.5 个工时，不满 20 分钟的那部分直接舍掉。

**举个例子**：
- 某人某天实际工时是 608 分钟
- `608 / 20 = 30.4`，向下取整得 30
- `30 × 0.5 = 15.0` 工时
- 如果实际是 617 分钟：`617 / 20 = 30.85`，向下取整还是 30，还是 15.0 工时，多出来的 9 分钟不算

### 5. 出勤天数

当天只要有打卡记录（≥1 次），就算出勤 1 天。哪怕实际算出来工时是 0，也算出勤。

### 6. 异常标记

以下三种情况当天会标淡红色背景：

- **跨天并单**：凌晨打卡被并回前一天
- **打卡次数异常**：当天只有 1 次或 2 次打卡（正常应该是 4 次或 2 次中午外出）
- **晚餐扣除**：当天触发了晚餐扣除

## 人员规则怎么存

每个人的三个开关（厂区住宿 / 弹性午休 / 晚餐扣除）存在 PostgreSQL 的 `employee_rule` 表里。

上传新的考勤文件时，系统会自动：
- 新出现的人 → 创建默认规则（任杰/王清玲/梁宗茂会按上面的预设自动开）
- 文件里没有了的人 → 删除他的规则
- 同一个人名字变了 → 只更新名字，开关不动

所以你改过一次某个人的开关，下次再导入新文件，他的设置不会丢。

## 导出格式

**Excel**
- 人员汇总 Sheet：按总工时降序
- 考勤汇总 Sheet：日历矩阵，没打卡的格子留空
- 每日明细 Sheet：逐日逐人的完整记录
- 所有 Sheet 都冻结首行、带边框、有斑马纹

**CSV**
- 只有每日明细，纯文本格式

## 本地跑起来

需要 Java 21、Node.js 20+、PostgreSQL 16。

```bash
# 后端
mvn spring-boot:run

# 前端（另开一个终端）
cd frontend
npm install
npm run dev
```

然后打开 `http://127.0.0.1:8080/app/index.html`，默认账号 `admin` / `AttendanceAdmin#2026`。

生产包：

```bash
cd frontend && npm run build
cd ..
mvn package -DskipTests
java -jar target/attendance-audit-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

---

## 线上部署

服务器是 Ubuntu + systemd + Nginx 反代。

```bash
# 更新 jar
scp target/attendance-audit-0.0.1-SNAPSHOT.jar server:/opt/attendance/attendance-audit.jar
ssh server "sudo systemctl restart attendance.service"
```

Nginx 里配一段反代到 `127.0.0.1:8080` 即可，不用额外处理静态文件，因为前端资源已经打进 jar 包里了。
